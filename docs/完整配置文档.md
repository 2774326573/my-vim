## 系统概述

### 核心特性

这是一个完整的离线 Vim 开发环境，支持：

- ✅ **C++ 开发**: clangd LSP + CMake + GDB 调试
- ✅ **Python 开发**: pylsp LSP + debugpy 调试
- ✅ **前端开发**: HTML/CSS/JavaScript LSP
- ✅ **Markdown 编辑**: 语法高亮 + 表格 + 预览
- ✅ **代码补全**: asyncomplete + LSP
- ✅ **代码片段**: snipMate (无需 Python)
- ✅ **可视化调试**: Vimspector (F5-F12)
- ✅ **智能导航**: 跳转定义、查找引用、大纲视图
- ✅ **Git 集成**: fugitive + gitgutter
- ✅ **模糊搜索**: FZF 文件查找
- ✅ **快捷键菜单**: which-key 自动提示

## 工作流程

### C++ 开发流程

1. **创建项目**
   ```powershell
   mkdir my-project && cd my-project
   mkdir src include build
   ```

2. **配置 CMake**
   ```cmake
   # CMakeLists.txt
   cmake_minimum_required(VERSION 3.10)
   project(MyApp)
   set(CMAKE_CXX_STANDARD 17)
   set(CMAKE_BUILD_TYPE Debug)
   set(CMAKE_EXPORT_COMPILE_COMMANDS ON)
   add_executable(myapp src/main.cpp)
   ```

3. **生成构建文件**
   ```powershell
   cd build
   cmake .. -G "MinGW Makefiles" -DCMAKE_EXPORT_COMPILE_COMMANDS=ON
   copy compile_commands.json ..
   ```

4. **编辑代码**
   ```powershell
   vim src/main.cpp
   ```
   - clangd 自动启动
   - `gd` 跳转定义
   - `gr` 查找引用
   - `Space+rn` 重命名

5. **编译**
   ```powershell
   cmake --build . --config Debug
   ```

6. **调试**
   - 在 Vim 中按 `F9` 设置断点
   - 按 `F5` 启动调试
   - `F10` 单步执行

### Python 开发流程

1. **创建项目**
   ```powershell
   mkdir my-python-project && cd my-python-project
   ```

2. **编辑代码**
   ```powershell
   vim main.py
   ```
   - pylsp 自动启动
   - 代码补全、错误检查

3. **运行**
   ```powershell
   C:\Users\jinxi\my-vim\tools\python38\python.exe main.py
   ```

4. **调试**
   - 创建 `.vimspector.json`
   - 按 `F9` 设置断点
   - 按 `F5` 启动

### Markdown 编辑流程

1. **创建文档**
   ```powershell
   vim document.md
   ```

2. **使用功能**
   - `h2<Tab>` - 创建二级标题
   - `Space+tm` - 启用表格模式
   - `||` - 创建表格
   - `code<Tab>` - 代码块
   - `Space+mp` - 预览（需手动下载预编译包）

3. **表格编辑**
   - 启用表格模式后自动对齐
   - Tab 在单元格间移动

## 常见问题

### Q1: clangd 找不到头文件？

**A**: 确保生成了 `compile_commands.json`：

```powershell
cmake .. -DCMAKE_EXPORT_COMPILE_COMMANDS=ON
copy build\compile_commands.json .
```

### Q2: 补全不工作？

**A**: 检查 LSP 状态：
```vim
:LspStatus
```

应该显示对应语言的服务器正在运行。

### Q3: 代码片段不展开？

**A**: 
1. 确认使用 `Tab` 键而不是 `Enter`
2. 检查 snipMate 是否加载: `:echo exists('g:loaded_snipMate')`
3. 查看片段文件: `snippets/` 目录

### Q4: which-key 菜单不显示？

**A**: 
1. 按空格后等待 1-2 秒
2. 检查配置: `:echo exists('g:loaded_which_key')`
3. 手动触发: `:WhichKey '<Space>'`

### Q5: Vimspector 断点不生效？

**A**:
1. 确保编译时使用 Debug 模式
2. 检查 `.vimspector.json` 中的程序路径
3. 确认 GDB 路径正确

### Q6: Markdown 预览不工作？

**A**: 
markdown-preview.nvim 需要手动下载预编译包：
1. 从 GitHub 下载 `markdown-preview-win-x64.zip`
2. 解压到 `data/plugged/markdown-preview.nvim/app/bin/`

### Q7: Python 格式化不工作？

**A**: 
检查格式化工具是否安装：
```powershell
tools\python38\python.exe -m pip list | findstr "autopep8 black yapf"
```

### Q8: 如何卸载插件？

**A**:
1. 从 `config/plugins.vim` 删除插件声明
2. 重启 Vim
3. 运行 `:PlugClean`

### Q9: Vim 启动很慢？

**A**:
1. 检查启动时间: `vim --startuptime vim_startup.log`
2. 禁用不需要的插件
3. 使用延迟加载: `Plug '...', { 'on': '...' }`

### Q10: 如何更新所有插件？

**A**:
```vim
:PlugUpdate
```

### Q11: 终端中文乱码？

**A**:
在 _vimrc 中确认：
```vim
set encoding=utf-8
set fileencoding=utf-8
set termencoding=utf-8
```

### Q12: 如何添加自定义代码片段？

**A**:
编辑 `snippets/<language>.snippets`：
```
snippet trigger
    代码内容
    ${1:占位符}
```
