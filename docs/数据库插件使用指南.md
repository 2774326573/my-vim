# vim-dadbod 数据库插件使用指南

## 功能介绍

vim-dadbod 是一个强大的数据库交互插件，支持多种数据库：
- ✓ MySQL/MariaDB
- ✓ PostgreSQL
- ✓ SQLite
- ✓ MongoDB
- ✓ Redis
- ✓ 以及更多...

## 快捷键

| 快捷键       | 功能               |
| ------------ | ------------------ |
| `<Space>db`  | 打开/关闭数据库 UI |
| `<Space>dba` | 添加数据库连接     |
| `<Space>dbf` | 查找数据库缓冲区   |
| `<Space>dbr` | 重命名缓冲区       |
| `<Space>dbl` | 查看最后查询信息   |

## 数据库连接配置

### 方法 1: 使用 g:dbs 全局变量

在 `config/plugin-config.vim` 中添加：

```vim
let g:dbs = {
\  'dev': 'mysql://user:password@localhost/mydb',
\  'staging': 'postgresql://user:password@localhost:5432/mydb',
\  'sqlite': 'sqlite:///path/to/database.db',
\}
```

### 方法 2: 使用 .env 文件

创建 `~/.config/db_ui/connections.json` 或项目根目录的 `.db_connections.json`:

```json
{
  "mysql_local": {
    "url": "mysql://root:password@localhost/mydb",
    "save_location": "~/.local/share/db_ui/queries"
  },
  "postgres_dev": {
    "url": "postgresql://user:pass@localhost:5432/devdb",
    "save_location": "./db_queries"
  }
}
```

## 数据库 UI 使用

1. **打开 UI**: 按 `<Space>db`
2. **导航**:
   - `o` 或 `<CR>` - 展开/折叠
   - `S` - 选择连接
   - `R` - 重新加载
3. **执行查询**:
   - 在 SQL 缓冲区按 `<Leader>S` 执行选中的 SQL
   - 或 `<Leader>SS` 执行整个文件

## 连接字符串格式

### MySQL
```
mysql://[user[:password]@]host[:port]/database
mysql://root:secret@localhost/mydb
mysql://localhost/mydb  # 无密码
```

### PostgreSQL
```
postgresql://[user[:password]@]host[:port]/database
postgresql://user:pass@localhost:5432/mydb
```

### SQLite
```
sqlite:///absolute/path/to/file.db
sqlite://./relative/path/to/file.db
```

### MongoDB
```
mongodb://[user:password@]host[:port]/database
mongodb://localhost:27017/mydb
```

## 在 Vim 中直接执行 SQL

### 方法 1: 可视化选择执行
```vim
" 1. 选择 SQL 语句（可视模式）
" 2. 输入命令
:'<,'>DB mysql://localhost/mydb
```

### 方法 2: 执行整个文件
```vim
:DB mysql://localhost/mydb < %
```

### 方法 3: 使用变量
```vim
:let g:db = 'mysql://localhost/mydb'
:'<,'>DB g:db
```

## 示例 SQL 文件

创建 `test.sql`:

```sql
-- 连接: mysql://root@localhost/test

-- 查询示例
SELECT * FROM users LIMIT 10;

-- 插入示例
INSERT INTO users (name, email) VALUES ('John', 'john@example.com');

-- 更新示例
UPDATE users SET status = 'active' WHERE id = 1;

-- 删除示例
DELETE FROM users WHERE status = 'inactive';
```

## 数据库补全

数据库补全已集成到 asyncomplete，支持：
- 表名补全
- 列名补全
- SQL 关键字补全

在 SQL 文件中输入时自动触发。

## 常用命令

```vim
" 执行当前行 SQL
:.DB

" 执行选中的 SQL
:'<,'>DB

" 执行整个文件
:%DB

" 使用特定连接执行
:DB mysql://localhost/mydb SELECT * FROM users;
```

## 配置选项

```vim
" UI 窗口位置
let g:db_ui_win_position = 'left'  " 或 'right'

" UI 窗口宽度
let g:db_ui_winwidth = 30

" 显示数据库图标
let g:db_ui_show_database_icon = 1

" 使用 NERDTree 风格图标
let g:db_ui_use_nerdtree = 0

" 强制显示通知
let g:db_ui_force_echo_notifications = 1
```

## 保存查询

查询会自动保存到配置的目录：
- 默认: `~/.local/share/db_ui/`
- 可自定义每个连接的保存位置

## 故障排除

### 1. 连接失败
检查：
- 数据库服务是否运行
- 用户名和密码是否正确
- 主机和端口是否正确

### 2. 无补全提示
确保：
- SQL 文件类型正确 (`:set ft=sql`)
- asyncomplete 正常工作
- 数据库连接成功

### 3. 查看日志
```vim
:messages
```

## 快速开始示例

### SQLite 快速测试

```vim
" 1. 创建测试数据库
:!sqlite3 test.db "CREATE TABLE users (id INTEGER, name TEXT);"
:!sqlite3 test.db "INSERT INTO users VALUES (1, 'Alice');"

" 2. 在 Vim 中查询
:DB sqlite:///test.db SELECT * FROM users;
```

### MySQL 快速测试

```vim
" 设置连接
:let g:db = 'mysql://root@localhost/test'

" 执行查询
:DB g:db SELECT DATABASE();
```
